Project Overview

I worked as Intermediate PHP developer in this project. The project aimed to provide a secure and efficient online business for lending and investment activities. Flow chart:

Role and Responsibilities

  • Developed core platform modules.
  • Continuously optimizied the platform functions and addressed challenging problems.
  • Worked closely with product managers, web designers, and testing engineers, to ensure timely project delivery.

Technologies and Tools Used

  • Front-end: HTML, CSS, JavaScript, jquery
  • Back-end: PHP, ThinkPHP framework,Phalcon framework, MySQL, MongoDB, Redis, RabbitMq
  • Version Control: Git
  • Security: Encryption techniques, authentication and authorization methods

Project Challenges and Solutions

  • Challenge: High quality demand to data privacy and security.
  • Solution: Using SSL/TLS protocol between browsers and servers, applied multiple encryption techniques such as AES, RSA, etc in API interaction.
  • Challenge: Server performance bottlenecks and response delays.
  • Solution: Implemented a distributed architecture, load balancing techniques and cache mechanism(memached, redis).
  • Challenge: The business is increasingly complex, and the maintenance cost of the system is rising.
  • Solution: Decoupled the business through microservices and using message queues to communicate between systems.

Self-Assessment

Adapting to the fast-paced, evolving nature of the industry has allowed me to continuously learn and innovate. I'm proud to contribute to systems that directly impact asset security and transaction processes for clients.
